<section class="features">
    <h2>Deployer Features</h2>
    <div class="grid -of3 -gutters">
        <div class="cell">
            <h3>Fast</h3>
            <p>
                Deployer is packed with time-saving features and optimisations, best for performance and
                development.
            </p>
        </div>
        <div class="cell">
            <h3>Modular</h3>
            <p>
                Create your own deployment script based on our modular blocks called "recipes".</p>
        </div>
        <div class="cell">
            <h3>Clean code</h3>
            <p>
                Deployer has very clean and well tested code. The repository is watched by many code quality tools, and
                every pull request is also checked by QA tools.
            </p>
        </div>
        <div class="cell">
            <h3>Rollbacks</h3>
            <p>
                If something is wrong, a simple rollback to the previous release is possible.
            </p>
        </div>
        <div class="cell">
            <h3>Atomic deploys</h3>
            <p>
                Prepare codebase, warm cache, do other stuff, and then deploy them with symlinks!
            </p>
        </div>
        <div class="cell">
            <h3>Parallel tasks</h3>
            <p>
                Deployer can run tasks in parallel, no other extension is required!
            </p>
        </div>
        <div class="cell">
            <h3>Consistency</h3>
            <p>
                Deployer keeps consistency between servers, even in parallel mode! If one task fails, subsequent
                tasks
                will not be executed on all servers.
            </p>
        </div>
        <div class="cell">
            <h3>Community Support</h3>
            <p>
                If you are having problems, you can ask for help on our <a href="/discuss">community chats</a>.
            </p>
        </div>
    </div>
</section>
